# noado (노아도) 프로젝트 핸드오버 컨텍스트
**AI 에이전트(Claude Code 등) 작업 이어가기를 위한 핵심 가이드 문서**

이 문서는 '대우 고시원' 및 소규모 개인 임대 사업자를 타겟으로 개발 중인 스마트 임대관리 솔루션 **noado(노아도)** 의 프로젝트 컨텍스트를 담고 있습니다. AI 봇에게 이 문서를 먼저 읽게 하면 기존 맥락을 잃지 않고 곧바로 작업을 이어갈 수 있습니다.

## 1. 프로젝트 주요 정보
*   **서비스명**: noado (노아도) - "매일 반복되던 임대관리 업무, 이제 AI 비서 노아도가 대신합니다."
*   **포지셔닝**: 무거운 ERP(예: 렌트플로우)가 부담스러운 **꼬마빌딩, 고시원, 오피스텔 10~50채 보유 개인 사업자**를 위한 가볍고 직관적인 구독형 SaaS.
*   **핵심 가치**: 압도적으로 쉬운 UI/UX (미니멀리즘), 임차인 전용 카카오톡 모바일 포털(앱 설치 X), 매일 아침 브리핑해 주는 AI 비서(Auto Agent).

## 2. 기술 스택 (Tech Stack)
*   **프레임워크**: Next.js (App Router, React 18+)
*   **스타일링**: Tailwind CSS (바닐라 CSS와 함께 사용), Lucide React (아이콘)
*   **데이터베이스 & 인증**: Supabase (PostgreSQL, Row Level Security 활용)
*   **배포 환경**: Vercel (예정)
*   **외부 연동**: 
    - 카카오 알림톡 (Solapi API 연동 완료)
    - 엑셀 입출금 내역 임포트 파싱 (`xlsx` 라이브러리)

## 3. 구현된 핵심 기능 (Done)
1.  **스마트 대시보드 및 다중 사업장 관리**
    *   `/dashboard`: 수납 현황, 이번 달 예정 수익, 미납금 등을 대시보드에서 시각화. 여러 건물 탭 전환 로직 작동.
    *   `/tenants`: 호실 상태(정상/미납/공실) 직관화. 임차인 정보 및 계약 약정일 관리.
2.  **임차인 전용 모바일 포털**
    *   `/portal/[tenantId]`: 임차인이 이메일 및 카카오톡 간편 로그인으로 접속하는 마이페이지. (인증 플로우 구축 완료)
    *   월세 결제 내역 확인, 증빙 다운로드 기능. 추가로 신용카드 가상 결제 데모 UI 구현됨.
3.  **수납 자동 매칭 알고리즘**
    *   `/payments`: 수기 통장 입출금 내역이나 엑셀 파일을 업로드하면, 입금자명과 금액을 기존 호실 계약 정보와 **자동으로 짝지어(매칭)** 수납 완료 및 미납 현황 동기화.
4.  **AI 비서 (Auto Agent)**
    *   임대인 대상 일일 스마트 브리핑 (카카오 알림톡 발송 기능 구축됨).
    *   납기일 경과된 세입자 대상 체납 독촉(Unpaid Reminder) 발송 트리거 시스템 연동.
5.  **마스터 관리자 페이월 시스템**
    *   `/master-admin`: 서비스 전반의 유저/사업자 가입 현황 및 결제 상태 모니터링 관리자 페이지 구축 완료.

## 4. 데이터/인증 구조 (Architecture)
*   **데모 환경 분기**: 현재 `AuthProvider` 및 `BusinessProvider`에서 로그인한 유저가 '데모 사용자'인 경우(`demo-user-123`), 실제 DB를 호출하지 않고 `src/lib/data.ts` 에 정의된 **가상 데이터(Mock Data)** 126건을 프론트엔드에 올려서 보여줍니다. (렌트플로우 분석 시 확인)
*   이로 인해 DB가 없어도 방문자가 랜딩 페이지에서 "지금 바로 체험하기"를 누르면 완벽하게 대시보드를 체험할 수 있습니다.

## 5. 앞으로 해야 할 일 (To-Do / Roadmap)
1.  **운영(Production) 배포**: 도메인(`noado.kr` 등) 연동을 통한 Vercel 정식 런칭.
2.  **카카오 승인 재심사**: Solapi 알림톡 템플릿의 버튼 연결 링크를 로컬호스트에서 정식 도메인으로 변경.
3.  **서비스 개발 배경 스토리텔링 (NEW)**: 랜딩 페이지 하단이나 별도 소개 페이지에 "왜 이 서비스를 만들게 되었는지"를 담은 텍스트/영상 아티클 기획. (마지막 추가 Task)

## 6. AI 에이전트 작업 지침서 (Prompt Rule)
이 프로젝트 코드를 수정하거나 새로운 기능을 만들 때, AI 봇은 다음 규칙을 **반드시** 따라야 합니다:
1.  **심미성 최우선**: 노아도는 다소 복잡한 백엔드보다 "압도적으로 깨끗하고 예쁜 UI"가 세일즈 포인트입니다. Tailwind를 사용해 모던하고 부드러운 글래스모피즘, 둥근 모서리, 미니멀한 여백을 극대화할 것.
2.  **경량화**: 복잡한 상태 관리(Redux 등)를 지양하고 React Hook(Provider)을 적극 이용할 것.
3.  **데모 데이터 무결성 체크**: 새 기능을 추가할 때, 실제 Supabase DB 연동뿐 아니라, `src/lib/data.ts`의 Mock 데이터 환경에서도 데모가 정상 작동하도록 분기(if) 처리할 것.
