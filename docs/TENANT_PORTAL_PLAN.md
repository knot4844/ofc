# 임차인 전용 포털 (Tenant Portal) 구축 계획

## 1. 목적 및 개요
현재 사이트는 로그인 시 무조건 마스터/임대인용 대시보드(`/dashboard`)로 연결됩니다. 하지만 임차인은 전체 사업장 매출을 보면 안 되며, 오직 본인의 **계약 정보**, **납부 내역**, **수납 계좌**만을 볼 수 있는 격리된 전용 페이지가 필요합니다. 본 계획서는 임대인(관리자)과 임차인 계정을 분리하고, 임차인 전용 포털을 구축하기 위한 가이드입니다.

## 2. 데이터베이스 및 권한(Role) 시스템 분리
- **유저 역할 부여**: Supabase User Metadata (또는 별도 `user_roles` 테이블)에 사용자의 유형을 `LANDLORD`(임대인)와 `TENANT`(임차인)로 구분하여 저장.
- **임차인 - 호실 매핑**: `rooms` 테이블에 해당 호실을 사용하는 임차인의 식별자(`tenant_email` 또는 Supabase Auth `user_id`)를 연결하여 권한을 증명.
- **보안 정책(RLS) 수정**: 임차인 계정은 전체 사업장(`businesses`) 데이터 접근이 불가하며, 오직 자신과 매핑된 `rooms` 및 `payments` 데이터만 조회할 수 있도록 RLS(Row Level Security) 정책을 강화.

## 3. 라우팅 및 로그인 분기 (Middleware 수정)
- **로그인 분기**: `/auth/callback` 및 미들웨어(`middleware.ts`)를 수정하여, 로그인 완료 시 사용자의 Role을 확인 후 접속 경로를 분기.
    - 임대인(`LANDLORD`) ➔ `/dashboard` 이동
    - 임차인(`TENANT`) ➔ `/tenant-portal` 이동
- **접근 제어(Guard)**: 임차인이 임대인 페이지(`/dashboard`, `/tenants`, `/reports` 등)에 URL로 강제 접속을 시도할 경우 에러 또는 홈으로 튕겨내는 방어 로직 추가.

## 4. 전용 UI/UX 구현 (`/tenant-portal`)
### 4.1. 임차인용 대시보드 화면 구성
- **GNB/레이아웃**: 좌측 메뉴(Sidebar) 없이 상단 네비게이션 바만 있는 심플하고 독립적인 레이아웃 사용.
- **계약 요약 위젯**: 계약 호실명, 계약 기간(시작~종료), 보증금/월세 금액, 매월 정기 납부일 표기.
- **결제 상태 위젯 (Alert)**: 미납된 금액이 있다면 붉은색 알림으로 [입금 계좌번호]와 [납부 필요 금액]을 최상단에 노출.
- **납부 내역 리스트**: 과거 월세/관리비 결제 내역을 리스트 형태로 최신순 제공.
- **전자계약서 조회**: 현재 서명하여 보관 중인 임대차 계약서 뷰어 및 다운로드 기능 (1차 개발 후 추후 확장).

### 4.2. 로그인 화면(UI) 개선 지원
- 기존 통합 로그인 화면(`/login`)은 유지하되, 필요 시 "임차인으로 시작하기 / 임대인으로 시작하기" 모드를 선택하여 가입/로그인 할 수 있는 UX로 고도화 가능.

## 5. 실행 순서 (Task List)
1. **[DB/권한]** Supabase에 `users` 롤(Role) 속성 추가 및 RLS 권한 수정.
2. **[Auth/Route]** Next.js 로그인 분기 로직 및 미들웨어 강화 구현.
3. **[UI/UX]** `/tenant-portal` 페이지 세트(홈, 내역조회, 설정 UI) 퍼블리싱.
4. **[API 연동]** 임차인 본인의 호실 정보만 불러오는 전용 API 엔드포인트 마련.
5. **[QA 및 테스트]** 실제 임차인 계정을 생성하고 대시보드 접근 통제 검증.
